# v0.4.1 Gzip Double-Decode Fix

**Issue:** [#769](https://github.com/teng-lin/notebooklm-py/issues/769) | **PR:** [#771](https://github.com/teng-lin/notebooklm-py/pull/771) | **Merged:** 2026-05-17

## Reproduction

Every batchexecute RPC fails — `LIST_NOTEBOOKS`, `CREATE_NOTEBOOK`, etc:

```
$ notebooklm list
RPC LIST_NOTEBOOKS server/network error (DecodingError); backing off 1.1s then retrying (1/3)
...
RPC LIST_NOTEBOOKS failed after ~8s: Error -3 while decompressing data: incorrect header check (server-error retries exhausted)
Network error: Request failed calling LIST_NOTEBOOKS: Error -3 while decompressing data: incorrect header check
```

`notebooklm doctor` passes. Raw `httpx.post` to batchexecute endpoint decodes fine. Only the library's streaming path fails.

## Root Cause

In `_core_transport.py`, `_stream_post_with_size_cap`:

1. Uses `response.aiter_bytes()` to read chunks — httpx **transparently decompresses** gzip, yielding raw bytes
2. Rebuilds a fresh `httpx.Response(status_code=200, headers=response.headers, content=bytes(buffer), ...)`
3. The copied `response.headers` still contain `Content-Encoding: gzip`
4. When downstream code accesses `response.text`, httpx's `Response.__init__` calls `self.read()` → `iter_bytes()` → gzip decoder on **already-decoded bytes** → `zlib.error: Error -3 while decompressing data: incorrect header check`

Minimal repro (no network):

```python
import httpx
httpx.Response(
    status_code=200,
    headers={"content-encoding": "gzip"},
    content=b'[["hello"]]',
)
# zlib.error: Error -3 while decompressing data: incorrect header check
```

## Fix (Patch)

File: `<venv>/lib/python3.11/site-packages/notebooklm/_core_transport.py`

Replace lines ~174-183 — the `return httpx.Response(...)` block in `_stream_post_with_size_cap`:

**Before:**
```python
        # Reconstruct a fully-buffered Response so downstream consumers
        # (``_core_rpc.py`` decode path) can use ``.text`` / ``.content``
        # without dealing with stream state. The request handle is carried
        # over so log/repr surfaces still point at the originating request.
        return httpx.Response(
            status_code=response.status_code,
            headers=response.headers,
            content=bytes(buffer),
            request=response.request,
        )
```

**After:**
```python
        # Reconstruct a fully-buffered Response so downstream consumers
        # (``_core_rpc.py`` decode path) can use ``.text`` / ``.content``
        # without dealing with stream state. The request handle is carried
        # over so log/repr surfaces still point at the originating request.
        #
        # The body bytes have already been decoded by httpx's streaming path
        # (``aiter_bytes()`` yields decompressed bytes).  If we copy headers
        # verbatim the downstream ``httpx.Response.__init__`` will see
        # ``content-encoding`` again and double-decode, raising
        # ``zlib.error: Error -3 while decompressing data: incorrect header check``.
        # Strip the spent encodings (and the now-mismatched content-length)
        # before materialising the buffered response.
        clean_headers = httpx.Headers(
            (k, v)
            for k, v in response.headers.items()
            if k.lower() not in ("content-encoding", "content-length")
        )
        return httpx.Response(
            status_code=response.status_code,
            headers=clean_headers,
            content=bytes(buffer),
            request=response.request,
        )
```
