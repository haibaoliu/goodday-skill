# SochDB — AI-Native Database for Agent Memory & Context

> Analyzed 2026-05-25 · 29 stars · Rust · 302K LOC · 458 files · AGPL-3.0
> https://github.com/sochdb/sochdb

## Summary

SochDB collapses vector DB + relational DB + context builder + graph memory into a single ~700KB embedded binary. Not a toy — 302K lines of Rust across a full database kernel (MVCC, WAL, columnar storage, HNSW, BM25, SIMD quantization).

## Architecture

```
sochdb-client (Rust/Python/Node/Go)  ← SDK
sochdb-query   (cost optimizer, context builder, SOCH-QL)
sochdb-kernel  (MVCC + WAL + catalog)
  ├── sochdb-storage (columnar LSCS + mmap)
  └── sochdb-index   (B-Tree + HNSW)
sochdb-vector  (SIMD, quantization, BM25, LSM) ← largest module
```

## Five Key Innovations

### 1. Knowledge Object — Co-Located Data Model
Data + edges + embeddings + temporal coordinates stored together. Fused query from ~11ms → ~300μs (30-50×). BLAKE3 content-addressed. Provenance chains. Bitemporal (valid time + system time).

### 2. ContextQuery Builder — Token Budget-Driven Assembly
Declarative sections (SYSTEM/USER/HISTORY/DOCS) with priority ordering. Token budget enforced with 4 truncation strategies. Outputs TOON format.

### 3. TOON Format — LLM-Optimized Serialization
Tabular header: `users[3]{id,name,email}:` then data rows. 67% token reduction vs JSON. Not compression — removes structural noise (quotes, braces, repeated keys).

### 4. Bitemporal + Provenance — Auditable Memory
Query `as_of(T₁).valid_at(T₂)` — "what did the system believe at T₁ about T₂?" Hash-linked derivation tracking.

### 5. Policy Engine — Database-Level Safety
Pre/post read/write hooks with Allow/Deny/Modify/Log actions. Regex key patterns. Per-agent rate limiting.
