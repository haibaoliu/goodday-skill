# 多角度追问 (Angle Interview)

**同一件事，换一个角度问就是另一把钥匙。角度之间的答案如果有矛盾，那里就是模糊点。**

此 Skill 在接到复杂/模糊任务时触发，不从单一维度追问，而是从多个对立角度各问一个问题，对比答案找到隐藏假设。

## 触发条件

以下情况可以主动加载此 Skill：
- 用户说"帮我做 X"但 X 很模糊
- 任务涉及多个子系统/多人/多步骤
- 用户之前有过"这不是我想要的"的返工经历
- 用户主动说"先 interview 一下"

触发时说：
> 角度追问：从 N 个角度确认一下？

## 角度库（6 组对立角度）

每组包含两个互补/对立的角度，从张力的两端提问：

### 组 1：望远镜 🔭 vs 显微镜 🔬
| 角度 | 问题 | 挖掘 |
|------|------|------|
| 🔭 望远镜 | "这件事最终最理想的样子是什么？" | 愿景、野心 |
| 🔬 显微镜 | "最让你头疼的一个细节是什么？" | 实现障碍、痛点 |

> 张力：愿景很大但细节说不清 → 方向对了但落地没想好

### 组 2：恐惧 😨 vs 删除 ✂️
| 角度 | 问题 | 挖掘 |
|------|------|------|
| 😨 恐惧 | "你最怕上线后发生什么？（说 1-3 个）" | 风险、盲区 |
| ✂️ 删除 | "如果必须砍掉一块，你砍哪个？" | 真正优先级 |

> 张力：最怕 X 但又愿意砍掉保护 X 的东西 → 认知不一致

### 组 3：回退 ⏪ vs 他人 👤
| 角度 | 问题 | 挖掘 |
|------|------|------|
| ⏪ 回退 | "如果只有一天，你只做哪一步？" | MVP 边界 |
| 👤 他人 | "别人（用户/同事/老板）会怎么吐槽这个？" | 盲区、沟通断层 |

> 张力：MVP 很小但没人会吐槽 → 可能真的够用；MVP 很大还有人吐槽 → 优先级没想清

### 组 4：理想 🎯 vs 现实 ⚙️
| 角度 | 问题 | 挖掘 |
|------|------|------|
| 🎯 理想 | "抛开所有限制，你真正想做成什么样？" | 去除约束后的真实需求 |
| ⚙️ 现实 | "有哪些限制是绝对不能碰的？" | 硬约束 |

> 张力：理想和现实的差距 = 需要妥协的空间

### 组 5：过去 📜 vs 未来 🔮
| 角度 | 问题 | 挖掘 |
|------|------|------|
| 📜 过去 | "之前试过类似的事吗？为什么没成？" | 历史教训 |
| 🔮 未来 | "做完这个，下一步是什么？" | 扩展性需求 |

> 张力：过去失败的坑和未来的扩展方向可能冲突

### 组 6：自己 🧍 vs 对手 🥊
| 角度 | 问题 | 挖掘 |
|------|------|------|
| 🧍 自己 | "你觉得这件事你最擅长哪个部分？" | 信心来源（可能过度自信） |
| 🥊 对手 | "如果让别人来做，他会在哪一步比你做得好？" | 自知短板 |

> 张力：自己擅长 X 但别人可能在 X 上做得更好 → 重新评估

## 使用方式

### 基础用法（3 角度，30 秒）

默认选 **恐惧 + 删除 + 回退**，这三个角度张力最强，覆盖风险、优先级、边界：

```
🔍 动手之前确认三件事：

😨 恐惧：你最怕上线后发生什么？
✂️ 删除：如果必须砍掉一块，砍哪个？
⏪ 回退：如果只有一天时间，你做哪一步？
```

### 进阶用法（按场景选角度组合）

| 场景 | 推荐角度组合 | 理由 |
|------|-------------|------|
| 新项目启动 | 望远镜 🔭 + 恐惧 😨 + 回退 ⏪ | 愿景、风险、MVP |
| 重构/改进 | 过去 📜 + 恐惧 😨 + 删除 ✂️ | 教训、新风险、取舍 |
| 安全相关 | 恐惧 😨 + 他人 👤 + 现实 ⚙️ | 威胁、攻击面、硬约束 |
| 团队协作 | 他人 👤 + 对手 🥊 + 回退 ⏪ | 预期管理、分工、交付 |
| 快速决策 | 删除 ✂️ + 回退 ⏪ | 最少问题，最快对准 |

## 矛盾检测

三个答案出来后，做一次比对：

```
矛盾检测：
1. 恐惧 vs 删除：你害怕 X，但你愿意砍掉 Y。Y 和 X 有关吗？
   如果有关 → ⚠️「你怕数据泄露，但愿意砍掉审计日志」= 认知矛盾
2. 恐惧 vs 回退：你在一天内做的步骤，能避免你最害怕的事吗？
   如果不能 → ⚠️ MVP 方向可能不对
3. 删除 vs 回退：你砍掉的和你优先做的是同一个东西的不同部分吗？
   如果是 → ⚠️ 删错了 / 优先错了
```

## 结果输出

```
🔍 角度追问结果

你的回答：
😨 恐惧：「最怕用户数据泄露」
✂️ 删除：「可以砍掉管理后台」
⏪ 回退：「先做登录和权限」

⚠️ 发现 1 个矛盾：
恐惧说要防泄露，但管理后台恰好是审计和权限管理的地方。
删掉管理后台 = 自断一臂。确认一下：是权限管理不用了，还是换个方式做？

✅ 2 个一致：
- MVP（登录+权限）能直接解决恐惧（泄露），方向正确
- 回退和恐惧高度一致，优先级没问题

建议：管理后台不要全砍，保留权限配置页面，砍掉数据报表/仪表盘部分。
```

## 迭代原则

- **每次用完记录**：什么角度有用？什么角度没用？什么角度应该加？
- **长期积累角度库**：不同场景有不同"黄金角度"
- **效果好就固化，不好就丢掉**：不为全面性保留无效角度
- **用户说"别问了直接做"就跳过**：工具是服务的，不是添乱的
