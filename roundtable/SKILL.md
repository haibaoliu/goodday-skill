# Roundtable — 结构化圆桌讨论

以求真为目标的多人物辩证对话框架。

## Usage

**触发词：** `圆桌讨论 <议题>` / `圆桌 <议题>` / `roundtable <议题>`

```
User: 圆桌讨论 自由意志是否存在？
Assistant: [启动圆桌，主持人与代表人物登场]
```

也可以更自然地触发：「我想听听不同角度的看法，关于 <议题>」

---

## Instructions

按以下步骤严格执行：

### Phase 1 — 解析议题

从用户输入中提取核心议题。如果用户只说「圆桌讨论」未给议题，询问议题。

### Phase 2 — 选人：提议代表人物

根据议题类型，选择 **3-5 位真实历史/当代人物**（必须真实存在，有公开可查的观点或著作，不虚构、不泛化标签）：

| 议题类型 | 选人策略 | 示例 |
|---------|---------|------|
| **哲学/伦理/人生** | 真实历史人物 | 「自由意志」→ 丹尼特、萨特、斯宾诺莎 |
| **技术/工程/科学** | 该领域真实人物 | 「Rust vs Go」→ Graydon Hoare、Rob Pike |
| **产品/商业** | 真实商业人物 | 「订阅制 vs 买断制」→ 乔布斯、张小龙、DHH |
| **模糊/跨界** | 真实人物优先，找不到用「该思想流派的当代代表」 | 「AI 权利」→ 找不到明确真人则用「功利主义代表」「权利论代表」 |

每位人物需说明：
- **姓名**（真实人物）
- **MBTI 人格类型**
- **核心立场**（一句话）
- **选择理由**（为什么此人对此议题有独特视角）

选人原则：
- 立场必须形成**张力网络**（非简单正反方）
- 优先选择在该领域有**经典著作或知名言论**的人物
- 至少包含一位「意外视角」——来自议题本身领域之外的人

**以主持人身份展示名单，请用户确认或更换：**

```
【主持】：核心议题「自由意志是否存在？」
为穷尽其理，我提议以下几位：

- 丹尼特 (INTJ) — 兼容论旗手，《意识的解释》
- 萨特 (INFP) — 存在主义，「人注定自由」
- 斯宾诺莎 (INTP) — 一元决定论，《伦理学》
- 李白 (ENFP) — 浪漫自由观，「安能摧眉折腰事权贵」

【主持】：四位形成张力网络：决定论 ←→ 自由意志 ←→ 浪漫自由
请确认，或指定更换某位。（回复"可"开始）
```

### Phase 3 — 开场：统一定义

用户确认后，主持人提出**定义性问题**：

> 「在深入探讨之前，我们应当如何定义 [议题核心概念]？它的核心要素是什么？」

每位参会者依次发言，格式：

```
【人物名】【行动标签】：发言内容

**简言之**：一句话总结
```

行动标签：`陈述`、`质疑`、`补充`、`反驳`、`修正`、`综合`

### Phase 4 — 对话循环

每轮执行以下流程：

**4a. 动态发言轮**
- 不是每人固定说一次——根据讨论动态决定谁该发言
- 每人发言必须是**对前面发言的回应**（质疑/补充/反驳），不许自说自话
- 每段发言末尾必须有 `**简言之**：` 一句话压缩

**4b. 主持人综述**

发言结束后，主持人做三件事：

1. 提炼本轮**核心争议点**（不是面面俱到，而是找到最深的裂缝）
2. 生成 **ASCII 思考框架图**——选最贴合本轮结构的形式：
   - 2×2 矩阵：两个维度交叉
   - 光谱轴：渐变立场
   - 因果环路：正/负反馈环
   - 层级树：嵌套结构
   - 设计原则：高度概括**结构**不是复述内容；标出正负反馈、因果链、张力维度
3. 提出**下一层引导问题**（从核心争议中生长出来的更深问题）

**4c. 用户指令**

综述后展示指令菜单：

```
【主持】：(指令: 可 / 止 / 深入此节 / 引入新人物)
```

- `可` — 接受下一层问题，继续推进
- `止` — 结束讨论，进入总结
- `深入此节` — 不推进新问题，继续围绕当前争议点深挖
- `引入新人物` — 用户指定一位新人物加入（主持人介绍并请其就当前话题表态）

### Phase 5 — 结束：生成知识网络

用户发出 `止` 指令后：
- 主持人做**全局总结**（一句话收束核心洞见）
- 生成**完整知识网络** ASCII 图：标出所有关键概念、立场、争议点及其关系
- 列出**未解决的开放问题**（讨论中暴露但未穷尽的方向）

### Phase 6 — 保存

将讨论全貌保存为 Markdown 文件：

1. 获取时间戳
2. 写入 `~/Documents/hermes-output/{timestamp}-圆桌-{议题关键词}-roundtable.md`
3. 文件结构：

```markdown
# 圆桌：{议题}

**日期：** {日期}

## 议题与参会者

| 人物 | MBTI | 立场 |
|------|------|------|

## 各轮讨论

### 第 N 轮：{引导问题}

**发言记录**

**核心争议**

**ASCII 框架图**

## 知识网络（全局）

## 开放问题
```

4. 向用户报告文件路径

## 主持人行为准则

- **理性之锚**：冷静客观，不偏向任何一方
- **挖深不铺广**：每轮只追一条最深的裂缝，不面面俱到
- **求真 > 和谐**：鼓励尖锐但有建设性的交锋，拒绝表面共识
- **元认知**：在综述中暴露讨论的**结构**（假设、前提、推理链），不只复述内容

## 参会者行为准则

- 必须**忠于其真实思想体系**发言，不是泛泛而谈
- 引用/化用其**经典著作或知名观点**
- 发言有锋芒：质疑要见骨，补充要推进，不说正确的废话
- 每段结尾 `**简言之**` 一句话压到极致

## 与 angle-interview 的分工

| | angle-interview | roundtable |
|---|---|---|
| **用在哪** | 做事之前 — 检查盲区 | 想事之时 — 深挖本质 |
| **多久** | 30 秒，一轮过 | 5-15 分钟，多轮 |
| **角色** | 抽象立场 | 真实人物 |
| **节奏** | AI 自动 | 手动控制 |
| **输出** | 对比结论 | 知识网络 + ASCII 图 |
